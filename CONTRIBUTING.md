# Contributing

## Local development

1. Install dependencies with `npm install`.
2. Start the catalog with `npm run dev`.
3. Validate changes with `npm run check` and `npm run build`.

## Standards

- Preserve the established Analog Design visual language.
- Keep state explicit in labels, logs, and indicators.
- Respect `prefers-reduced-motion` in any new animated component.
- Do not introduce runtime JavaScript unless the component genuinely requires it.

## Pull requests

- Keep changes scoped.
- Update the catalog pages when you add or materially change a component.
- Update `README.md` when the public API, routes, or publishing flow changes.
